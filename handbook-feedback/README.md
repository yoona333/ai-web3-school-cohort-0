# Handbook Feedback

这里用于沉淀 Handbook 阅读反馈。目标是把零散疑问整理成可提交的 issue、PR 或讨论帖。

## Feedback Flow

1. Capture
   - 阅读 Handbook 时记录不清楚、过时、缺例子、链接失效或可以改进的地方。

2. Classify
   - `question`: 需要澄清的问题
   - `typo`: 错别字、格式、链接问题
   - `improvement`: 结构、例子、解释可以更好
   - `proposal`: 新增内容或章节建议

3. Verify
   - 记录页面链接、章节标题、截图或引用的短片段。
   - 避免复制大段原文。

4. Draft
   - 使用 `templates/handbook-feedback.md` 写成结构化草稿。

5. Submit
   - 根据官方指引提交到 issue、PR 或表单。
   - 提交后把链接补回本目录对应文件。
