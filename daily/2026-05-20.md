## Reflection

The workflow makes the agent useful but bounded. It can help plan, explain, draft, organize, and check work, while human confirmation remains required for repo changes, commits, pushes, WCB submissions, and any account or platform state changes.

For Web3 safety, the agent should never touch API keys, tokens, private keys, seed phrases, wallet signatures, transfers, approvals, or contract writes. Its job is to prepare explanations and checklists, not to operate sensitive assets.
